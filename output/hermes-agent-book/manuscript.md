# 第3章：它是怎么工作的

> 不需要懂技术也能理解它的"大脑结构"。

## 3.1 四层架构，简单理解

把 Hermes Agent 想象成一栋四层楼：

```
┌─────────────────────────────────────────┐
│          第 4 层：入口大厅               │
│   Telegram │ Discord │ Slack │ 微信 │…   │
│   你从任何一个门都能进来                    │
├─────────────────────────────────────────┤
│          第 3 层：前台接待               │
│   负责识别你是谁、分配房间、处理定时任务    │
├─────────────────────────────────────────┤
│          第 2 层：AI 大脑               │
│   理解你说的话 → 选择模型 → 制定计划      │
│   → 执行操作 → 记住学到的经验             │
├─────────────────────────────────────────┤
���          第 1 层：干活的手脚             │
│   在服务器上执行命令、操作文件、浏览网页    │
└─────────────────────────────────────────┘
```

**第 4 层（消息平台）**：这是你接触 Hermes 的入口。你可以在 Telegram、Discord、Slack 等 15+ 平台上和它对话。

**第 3 层（消息网关）**：相当于"前台"。它负责：
- 认出你是谁（每个用户独立的会话）
- 处理定时任务（你设置的每天 9 点检查服务器）
- 安全检查（危险操作需要你批准）
- 把不同平台的"语言"翻译成统一的格式

**第 2 层（AI 核心）**：这是 Hermes 的"大脑"。它包含：
- **模型路由器**：选择用哪个 AI 模型来处理你的请求
- **上下文管理**：管理对话历史，太长时自动压缩
- **记忆系统**：跨会话记住重要信息
- **技能引擎**：管理和执行学到的技能
- **学习循环**：从经验中提取和改进技能

**第 1 层（执行终端）**：这是 Hermes 的"手和脚"。它可以在多种环境中执行命令，包括你的本地电脑、Docker 容器、远程服务器或云端。

## 3.2 学习循环：它是怎么"变聪明"的

这��� Hermes Agent 最独特的功能。简单说，它有一个四步学习闭环：

```
你给它一个任务
    ↓
它完成这个任务
    ↓
它把这个任务提炼成"技能模板"保存起来
    ↓
下次遇到类似任务，直接调用技能（更快！）
    ↓
如果���能执行中发现问题，自动优化技能
    ↓
重要知识写入记忆（下次对话还记得）
    ↓
回到起点，不断循环
```

> **打个比方**：就像一个新员工，第一天你手把手教他怎么做报表，他做完之后自己总结了一份"SOP（标准操作流程）"。第二天再做同样的报表，他直接照着 SOP 来，又快又准。如果发现 SOP 有问题，他会自己修改。时间久了，他就变成了这方面的专家。

## 3.3 模型路由：怎么选 AI 大脑

Hermes Agent 不绑定任何一个 AI 模型。它支持 14 家 AI 服务商、200 多个模型，你可以随时切换。

**它是怎么做到的？**

想象你有一张"万能SIM卡"——不管你用的是移动、联通还是电信的套餐，插上就能用。Hermes 的模型路由器就像这张万能SIM卡，统一了不同 AI 模型的调用方式。

**万一当前模型出问题怎么办？**

它有自动故障转移机制——如果主模型（比如 Claude）太忙或报错了，它会自动切换到备用模型（比如 GPT），你甚至感觉不到中断。

> **实际效果**：你在用 Claude 和 Agent 聊天，突然 Claude 的 API 限流了。Hermes 自动切换到 GPT 继续回答，回答完后下一轮又自动切回 Claude。整个过程对你���明。

## 3.4 上下文管理：对话太长怎么办

和 AI 聊久了，对话历史会越来越长，最终超出 AI 模型的"记忆容量"。这时候 Hermes 会自动压缩历史对话，只保留最重要的信息。

> **打个比方**：就像你看一本书，看到第 100 页时忘了前面的内容。Hermes 会自动帮你做"读书笔记"——把前面的核心内容压缩成摘要，这样你既有上下文，又不会超负荷。

它还会做一件事：**保护最近的几轮对话不被压缩**，确保对话的连贯性。

# 第5章：怎么安装和使用

> 从零开始，5 分钟上手。

## 5.1 安装前需要知道的事

> **注意**：Hermes Agent 目前不支持 Windows 系统。如果你用 Windows，需要通过 WSL2（Windows 子系统）来使用。Mac 和 Linux 用户可以直接安装。

**最低配置**：
- 一台 Mac 或 Linux 电脑（或服务器）
- 能上网
- 一个 AI 模型的 API Key（推荐 OpenRouter，注册就送免费额度）

## 5.2 一键安装

打开终端（Mac 上叫"终端"，Linux 上叫 Terminal），输入一行命令：

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

安装器会自动帮你搞定所有依赖（Python、Node.js 等），你只需要跟着提示走。

## 5.3 初次配置

安装完成后，运行设置向导：

```bash
hermes setup
```

设置向导会引导你：
1. 选择 AI 模型服务商（推荐选 OpenRouter）
2. 输入 API Key
3. 配置你想用的消息平台（Telegram、Discord 等）

> **小贴士**：如果你只是想先体验一下，配置好模型后直接在终端里输入 `hermes` 就能开始聊天了，不需要配置消息平台。

## 5.4 开始使用

**终���模式**（最简单）：

```bash
hermes
```

直接在终端里和 AI 聊天。

**网关模式**（连接多个平台）：

```bash
hermes gateway
```

启动后，Hermes 会同时在所有你配置好的平台上待命。

**常用命令**：

| 命令 | 功能 |
|------|------|
| `/new` | 开始新对话 |
| `/model` | 切换 AI 模型 |
| `/stop` | 中断当前任务 |
| `/browser` | 打开浏览器 |
| `/skills` | 查看可用技能 |

## 5.5 用 Docker 部署（进阶）

如果你想把 Hermes 部署在服务器上长期运行，可以用 Docker：

```bash
docker run -v ~/.hermes:/root/.hermes hermes-agent gateway
```

> **小贴士**：最低配的云服务器（1 核 1G，大约每月 5 美元）就能跑 Hermes Agent。

1. Mac/Linux 一键安装，Windows 需用 WSL2
2. `hermes setup` 引导配置，3 步上手
3. 终端模式适合体验，网关模式适合长期使用

Hermes Agent 从 2026 年 3 月才公开，到现在还不到一个月。它更新很快（平均 3-4 天一个版本），但快速迭代也意味着：

- 功能变化频繁，文档可能跟不上
- Bug 修复的同时可能引入新 Bug
- 部分功能还没有经过充分测试

> **打个比方**：它就像一个刚开业一个月的餐厅，菜品丰富、上菜快，但偶尔会端错盘子。

## 7.2 "会学习"的效果还有争议

这是 Hermes Agent 最大的卖点，但也是最大的争议点。

**理想情况**：Agent 从经验中自动学会技能，越用越顺。

**实际情况**：有用户反馈 Agent 会"忘记"自己有哪些能力。比如明明有终端访问权限，却反复问用户要权限。

> **客观评价**："学习"功能的设计思路是好的，但目前缺乏定量数据来证明它真的在有效改进。它更像是一个"经验笔记本"，而不是真正的"智能进化"。

## 7.3 上下文管理是最大痛点

> **数据**：和"上下文管理"相关的问题有 322 个，是用户反馈最多的问题类别。

简单说，当对话太长时，AI 的"记忆容量"不够用了。虽然 Hermes 有自动压缩机制，但压缩后可能会丢���重要信息，导致 Agent "答非所问"或"前言不搭后语"。

## 7.4 安全方面需要注意

- **危险操作默认需要你批准**，这是好的设计
- 但在 Docker 容器模式下，审批会被跳过（认为容器本身就是隔离的）
- 安全扫描服务如果出问题，默认会放行所有操作

> **建议**：如果你打算在正式环境使用，建议做额外的安全加固。

## 7.5 不适合这些场景

| 场景 | 原因 |
|------|------|
| 金融、医疗等关键领域 | 准确性没有保障，缺乏审计追踪 |
| 需要 Windows 原生支持 | 只能通过 WSL2 使用 |
| 多个 Agent 协同工作 | 不支持原生的多 Agent 编排 |
| 需要官方技术支持 | 纯社区驱动，没有商业支持 |
| 对稳定性要求极高 | 914 个待解决问题，仍在快速迭代中 |

## 7.6 一些小问题

- 安装脚本偶尔会出错
- 配置工具可能会覆盖你之前的设置
- 部分 AI 模型的兼容性不太好
- 树莓派等 ARM 设备缺少官方支持

---

**本章要点**

1. 项目还不到一个月大，914 个待解决问题，成熟度有待提升
2. "学习"功能的设计理念好，但实际效果缺乏数据验证
3. 上下文管理是用户反馈最多的问题（322 个相关 Issue）
4. 安全设��有亮点（审批系统）也有隐患（Docker 模式跳过审批）
5. 不适合关键业务场景，但适合个人使用和尝鲜

## 8.3 怎么选

> **选 Hermes Agent 如果你**：想要一个免费的、不被锁定在某个 AI 公司的、能跨平台使用的全能 AI 助手，而且不介意它偶尔有小毛病。

> **选 ChatGPT 如���你**：想要最简单、最稳定的体验，愿意付费，不需要高级功能。

> **选 Cursor 如果你**：主要需求是写代码，而且想要最好的编码体验。

> **选 CrewAI 如果你**：需要多个 AI Agent 协同工作（比如一个负责搜索、一个负责写作、一个负责审核）。

**本章要点**

1. 免费开源 + 会学习 + 多平台是 Hermes 的核心差异化
2. 稳定性和成熟度不如商业产品（ChatGPT、Claude Code）
3. 多 Agent 协作能力不如 CrewAI
4. 适合个人用户和尝鲜者，不适合对稳定性要求高的企业场景

# 第9章：未来会怎样发展

> 基于数据趋势的合理推测，不是官方承诺。

## 9.1 短期可能发生的事（1-2 个月）

- **更多 AI 模型支持**：每个版本都在加新模型，这个趋势会继续
- **Windows 原生支持**：用户需求强烈，可能会实现
- **技能市场更完善**：目前已有社区技能市场，可能会进一步丰富

## 9.2 中期可能发生的事（3-6 个月）

- **多个 Agent 协作**：目前只支持单个 Agent，但已有"子Agent"功能作为基础
- **企业级功能**：多实例隔离、Docker 部署已就位，可能会加权限管理和审计日志
- **可视化控制面板**：目���主要通过命令行管理，未来可能有网页界面

## 9.3 行业大趋势

1. **AI Agent 框架越来越多**：2026 年涌现了大量类似产品，市场竞争激烈
2. **从"只能写代码"到"什么都能干"**：Agent 正在从编程工具变成通用助手
3. **不被 AI 公司锁定**：用户越来越希望自由选择 AI 模型，而不是被绑定
4. **安全越来越重要**：AI Agent 有执行权限，安全问题不容忽视
5. **MCP 成了行业标准**：一个叫"模型上下文协议"的技术正在被广泛采用，Hermes 的实现是最完整的之一

## 9.4 一个大胆的预测

以当前的增长速度（27 天 43,000 Star），Hermes Agent 很可能在 3-6 个月内达到 **100,000 Star**。

但增长快也意味着挑战大——快速迭代可能引入新问题，社区期望越来越高，团队能否跟上是一个问号。

---

**本章要点**

1. 短期：更多模型、Windows 支持、技能市场完善
2. 中期：多 Agent 协作、企业功能、可视化面板
3. 行业趋势：通用化、模型自由、安全优先、MCP 标准化
4. 增长潜力大，但能否持续取决于团队能否跟上节奏
